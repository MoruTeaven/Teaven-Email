## 分类路由权限重构 — 从租户自管理改为超管统一管理

**核心变更：** 分类路由（category_routes）从租户可自行增删，改为超管后台统一配置，租户只读。

### 修改文件清单

1. **`src/routes/providers.ts`** — 移除 POST/DELETE 分类路由端点
   - 删除 `POST /v1/providers/routes` 和 `DELETE /v1/providers/routes/:id`
   - 保留 `GET /v1/providers` 和 `GET /v1/providers/routes`（租户只读）

2. **`src/routes/admin.ts`** — 新增超管分类路由管理 API
   - `GET /v1/admin/routes` — 获取所有分类路由（JOIN users/providers/accounts）
   - `POST /v1/admin/routes` — 为指定租户创建分类路由（校验 user+provider+account 存在性及账号归属）
   - `DELETE /v1/admin/routes/:id` — 删除分类路由

3. **`src/db.ts`** — 新增超管用数据访问方法
   - `getAllCategoryRoutesAdmin()` — 联表查询所有路由（带用户名/邮箱/通道名/账号邮箱）
   - `deleteCategoryRouteById(id)` — 按 ID 删除（不校验 user_id）
   - `getCategoryRouteById(id)` — 按 ID 查询

4. **`src/dashboard_html.ts`** — 租户端分类路由改为只读
   - 删除"分类路由"tab 页及 `switchProviderTab()`、`showRouteModal()`、`deleteRoute()` 函数
   - 发送通道页只展示通道卡片，底部以只读列表展示当前生效的分类路由规则
   - 标注"由管理员统一配置，你当前生效的路由规则"

5. **`src/admin_html.ts`** — 超管新增分类路由管理界面
   - 侧边栏新增"分类路由"导航项（fas fa-code-branch 图标）
   - `renderRoutes()` — 列表展示所有路由（用户+分类+通道+账号+优先级）
   - `showAddRouteModal()` — 弹窗表单：选租户→选分类→选通道→选账号（按通道过滤）→优先级
   - `onRouteProviderChange()` — 通道切换时动态过滤可选账号列表
   - `deleteAdminRoute()` — 删除确认弹窗
