## Bug 修复：POST /v1/setup/login 返回 404 "No API keys found"

**问题：** 用户密码正确但 api_keys 表中无记录时，login 端点返回 404 错误，前端无法进入 `/key-from-password` 创建新 Key，形成死锁。

**根因：** `src/routes/setup.ts` 第 151-153 行在 `apiKeys.length === 0` 时直接返回 404。前端 `dashboard_html.ts` 和 `admin_html.ts` 依赖 `resp.success === true` 才会调用 `key-from-password`。

**修复（3 个文件）：**

1. **`src/routes/setup.ts`** (149-164行) — 移除两个 404 检查（无 key / 无活跃 key），改为返回 `success: true` 含空 api_keys 数组
2. **`src/dashboard_html.ts`** (1039-1055行) — 登录成功后始终调用 `key-from-password`，不再检查活跃 key 是否存在
3. **`src/admin_html.ts`** (1535-1556行) — 同上

**未部署** — 本地无 CLOUDFLARE_API_TOKEN，需用户手动 `wrangler deploy`。
