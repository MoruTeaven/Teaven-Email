## 21:43 - 修复 /v1/dashboard/overview 500 错误

**问题**：访问 `GET /v1/dashboard/overview` 返回 HTTP 500。

**根因分析**：handler 中 5 个 D1 查询用 `Promise.all` 并发执行，任一失败则整体崩溃且无错误日志。最可能触发点是 `getApiKeysByUser` 依赖 migration 005 添加的 `auto_created` 列 — 若未执行迁移则 `WHERE auto_created = 0` 直接报 "no such column"。

**修复内容（3 个文件）**：

1. **`src/index.ts`** — 新增 `app.onError` 全局错误处理器，捕获未处理异常并返回结构化 JSON + console.error 日志。

2. **`src/routes/dashboard.ts`** — `Promise.all` → `Promise.allSettled`，每个查询独立容错，失败时记录到 `errors` 字段返回给前端，不影响其他查询。响应新增可选 `errors` 字段标识哪些数据源加载失败。

3. **`src/db.ts`** — `getApiKeysByUser` 增加 try/catch 兜底：若 `auto_created` 列不存在则降级到无过滤查询并 console.warn 提示执行 migration 005。

**部署后效果**：
- 即使部分查询失败，dashboard 仍返回可用数据 + 错误详情
- 全局 error handler 确保所有未处理异常返回有意义的 JSON（而非裸 500）
- `getApiKeysByUser` 在有/无 migration 005 的环境下均可工作
